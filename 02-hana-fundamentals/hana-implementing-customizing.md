## Steps

# Project preparation
- Cost and scope definition
- Compiling required interfaces
- Blueprinting

# Realization
- Configuration: match HANA to company business
- Customization: change not supported by SAP funcionality (must be low number)

# Testing
- Unit testing: individual components (e.g.: transactions are working?)
- Integration testing: data must flow correctly (e.g. data sales order => data accounts recievables)
- User acceptance testing: end-users test
- Performance testing: all data can be handled by system transactions.

# Final preparation
- Master data uploads
- User security roles
- End-user training

# Go-live
- End-users start using production daily

# Post-go-live support
- Issues raised by users are addressed
- Non-business-critical customizations are developed, tested and implemented
